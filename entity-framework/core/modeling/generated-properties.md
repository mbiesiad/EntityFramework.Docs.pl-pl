### <a name="no-value-generation"></a><span data-ttu-id="662b2-109">Brak generowania wartości</span><span class="sxs-lookup"><span data-stu-id="662b2-109">No value generation</span></span>

<span data-ttu-id="662b2-110">Generowanie wartości nie oznacza, że zawsze będzie podasz prawidłową wartość, która zostanie zapisana w bazie danych.</span><span class="sxs-lookup"><span data-stu-id="662b2-110">No value generation means that you will always supply a valid value to be saved to the database.</span></span> <span data-ttu-id="662b2-111">Ta prawidłowa wartość musi być przypisana do nowych jednostek przed dodaniem ich do kontekstu.</span><span class="sxs-lookup"><span data-stu-id="662b2-111">This valid value must be assigned to new entities before they are added to the context.</span></span>

<span data-ttu-id="662b2-114">W zależności od używanego dostawcy bazy danych wartości mogą być generowane po stronie klienta przez EF lub w bazie danych programu.</span><span class="sxs-lookup"><span data-stu-id="662b2-114">Depending on the database provider being used, values may be generated client side by EF or in the database.</span></span> <span data-ttu-id="662b2-115">Jeśli wartość jest generowana przez bazę danych, EF może przypisać wartość tymczasową podczas dodawania jednostki do kontekstu.</span><span class="sxs-lookup"><span data-stu-id="662b2-115">If the value is generated by the database, then EF may assign a temporary value when you add the entity to the context.</span></span> <span data-ttu-id="662b2-116">Ta wartość tymczasowa zostanie następnie zastąpiona wartością wygenerowaną przez bazę danych podczas `SaveChanges()`.</span><span class="sxs-lookup"><span data-stu-id="662b2-116">This temporary value will then be replaced by the database generated value during `SaveChanges()`.</span></span>

<span data-ttu-id="662b2-117">W przypadku dodania jednostki do kontekstu, który ma wartość przypisaną do właściwości, EF spróbuje wstawić tę wartość zamiast generować nową.</span><span class="sxs-lookup"><span data-stu-id="662b2-117">If you add an entity to the context that has a value assigned to the property, then EF will attempt to insert that value rather than generating a new one.</span></span> <span data-ttu-id="662b2-118">Właściwość jest uznawana za przypisaną wartości, jeśli nie ma przypisanej wartości domyślnej środowiska CLR (`null` dla `string`, `0` dla `int`, `Guid.Empty` dla `Guid`itd.).</span><span class="sxs-lookup"><span data-stu-id="662b2-118">A property is considered to have a value assigned if it is not assigned the CLR default value (`null` for `string`, `0` for `int`, `Guid.Empty` for `Guid`, etc.).</span></span> <span data-ttu-id="662b2-119">Aby uzyskać więcej informacji, zobacz [jawne wartości dla wygenerowanych właściwości](../saving/explicit-values-generated-properties.md).</span><span class="sxs-lookup"><span data-stu-id="662b2-119">For more information, see [Explicit values for generated properties](../saving/explicit-values-generated-properties.md).</span></span>

> [!WARNING]

>
> <span data-ttu-id="662b2-122">Na przykład podczas korzystania z SQL Server wartości będą generowane automatycznie dla `GUID` właściwości (przy użyciu algorytmu sekwencyjnego identyfikatora GUID SQL Server).</span><span class="sxs-lookup"><span data-stu-id="662b2-122">For example, when using SQL Server, values will be automatically generated for `GUID` properties (using the SQL Server sequential GUID algorithm).</span></span> <span data-ttu-id="662b2-123">Jeśli jednak określisz, że właściwość `DateTime` jest generowana przy dodawaniu, należy skonfigurować sposób generowania wartości.</span><span class="sxs-lookup"><span data-stu-id="662b2-123">However, if you specify that a `DateTime` property is generated on add, then you must setup a way for the values to be generated.</span></span> <span data-ttu-id="662b2-124">Aby to zrobić, należy skonfigurować wartość domyślną `GETDATE()`, zobacz [wartości domyślne](relational/default-values.md).</span><span class="sxs-lookup"><span data-stu-id="662b2-124">One way to do this, is to configure a default value of `GETDATE()`, see [Default Values](relational/default-values.md).</span></span>
